# 기획 리뷰 가이드

## 왜 기획 리뷰가 필요한가

코드 리뷰 봇은 "코드가 맞냐"만 봅니다. 하지만 진짜 중요한 질문은:

- 이 기능이 PRD에 있는 건가?
- 타겟 유저에게 가치가 있나?
- 이번 스프린트 목표와 맞나?
- 기존 서비스 흐름과 충돌하지 않나?

이건 봇이 못 합니다. **Claude Code가 리뷰 종합 단계에서 담당합니다.**

## Claude Code 기획 리뷰 체크리스트

리뷰 종합 코멘트 작성 시, 코드 리뷰 외에 아래 항목을 추가로 검토합니다:

### 1. PRD 일치 여부
```
- .project/PRD.md 를 읽고 이 PR의 기능이 PRD에 명시되어 있는지 확인
- PRD에 없는 기능이면: "PRD에 없는 기능입니다. PRD 업데이트 또는 기능 제거 필요"
- PRD와 다르게 구현했으면: "PRD에는 X로 되어있지만 Y로 구현됨. 의도된 변경인지 확인 필요"
```

### 2. 서비스 방향성
```
- CLAUDE.md의 프로젝트 개요/타겟 유저/핵심 가치를 기준으로 판단
- "이 변경이 타겟 유저에게 가치를 주는가?"
- "핵심 차별점을 강화하는가, 약화하는가?"
```

### 3. UX 흐름
```
- 새 기능이 기존 사용자 흐름을 깨뜨리지 않는지
- 모바일 퍼스트인 프로젝트에서 데스크톱만 고려한 구현은 아닌지
- 에러 상태, 로딩 상태, 빈 상태 처리가 되어 있는지
```

### 4. PRD 업데이트 필요 여부
```
- 스펙이 변경된 PR이면: "PRD 업데이트가 필요합니다" 코멘트
- 같은 PR에서 PRD도 함께 수정하는 것을 권장
- PRD 수정 없이 머지하면 드리프트 발생
```

## 리뷰 종합 코멘트 예시

```markdown
## 리뷰 종합 (Claude Code)

### 코드 리뷰 (봇 의견 종합)
| 지적 | Gemini | CodeRabbit | Copilot |
|------|:------:|:----------:|:-------:|
| 에러 핸들링 누락 | ✅ | ✅ | - | - |
| 타입 안전성 | - | ✅ | ✅ | - |

### 기획 리뷰
- ✅ PRD 일치: `.project/PRD.md` Phase 1 "실시간 시세" 항목에 해당
- ⚠️ PRD 차이: PRD에는 "10종목"이지만 52종목으로 확장 구현 → PRD 업데이트 필요
- ✅ 타겟 유저: 투자 정보를 빠르게 확인하려는 직장인에게 가치 있음
- ✅ UX: 로딩/에러/빈 상태 모두 처리됨

### 최종 판단
코드 수정: 에러 핸들링 1건 반영
PRD: 종목 수 52개로 업데이트 필요 → 같은 PR에서 수정 권장
```

## 봇에게 맥락을 주는 방법

봇에게 PRD 전문을 주입하면 outdated 문제가 생깁니다. 대신 **목표 한 줄 + 핵심 원칙**만 줍니다:

### .coderabbit.yaml
```yaml
language: "ko-KR"
reviews:
  instructions: |
    이 프로젝트의 목표: [한 줄 설명]
    코드 리뷰 시 위 목표와 명백히 충돌하는 변경이 있으면 지적해주세요.
```

> 기획 적합성의 상세 판단은 Claude Code에게 맡기고, 봇에게는 "명백한 충돌"만 잡도록 합니다.
