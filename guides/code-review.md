# 코드 리뷰 가이드

## 봇 4명 자동 리뷰 체계

PR이 열리면 4명의 AI 봇이 자동으로 코드 리뷰를 수행합니다.

### 설정 파일

| 파일 | 역할 | 위치 |
|------|------|------|
| `.github/workflows/pr_agent.yml` | Qodo PR Agent 실행 | 레포 루트 |
| `.github/workflows/label-pr.yml` | Copilot 리뷰 요청 + 자동 라벨링 | 레포 루트 |
| `.github/labeler.yml` | 경로 기반 라벨 규칙 | 레포 루트 |
| `.pr_agent.toml` | Qodo 상세 설정 | 레포 루트 |
| `.coderabbit.yaml` | CodeRabbit 언어 설정 | 레포 루트 |

### 봇별 리뷰 포인트

**Gemini Code Assist**
- 전체적인 코드 품질 평가
- 로직 오류, 엣지 케이스 지적
- 리팩토링 제안

**Qodo PR Agent**
- 보안 취약점 스캔
- 테스트 커버리지 확인
- 개선 제안 (committable suggestion)
- Ticket compliance 체크 (Issue 연결 시)

**CodeRabbit**
- 변경 사항 요약 (Walkthrough)
- 파일별 상세 리뷰
- Actionable comments with 코드 제안

**Copilot**
- 인라인 코드 제안
- 간결한 전체 리뷰

### Qodo 프로젝트 맥락 주입

`.pr_agent.toml`의 `extra_instructions`에 프로젝트 목표를 넣으면 봇이 맥락을 이해합니다:

```toml
[pr_reviewer]
extra_instructions = """
- 한국어로 리뷰 작성
- 보안 취약점 우선 확인
- 이 프로젝트는 [서비스 한 줄 설명]
- 핵심 원칙: [원칙 1], [원칙 2], [원칙 3]
"""
```

> 주의: PRD 전문을 넣지 마세요. 업데이트 안 되면 오히려 해가 됩니다. 목표 한 줄 + 원칙 3개 수준으로 유지하세요.

### 라벨 자동화

PR 본문에 "Generated by Claude Code"가 포함되면 자동으로 `ai-generated` + `claude-code` 라벨이 붙습니다. 경로 기반 라벨(`frontend`, `backend`, `docs`)도 자동 부착됩니다.
