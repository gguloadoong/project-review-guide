# 리뷰 토론 절차

## 전체 흐름

```
PR 생성
  → 최소 2분 대기
  → 봇 4명 리뷰 확인
  → "리뷰 종합" 코멘트 작성 (코드 + 기획)
  → @coderabbitai + /ask 동시 질문
  → 최소 2분 대기
  → 봇 응답 확인
  → "최종 판단" 코멘트 작성
  → 머지
```

## 토론 종료 규칙 (무한 루프 방지)

| 규칙 | 설명 |
|------|------|
| **최대 1회전** | 종합 코멘트(1회) → 봇 응답(1회) → 최종 판단 → 머지 |
| **추가 질문 금지** | 봇 응답에 새 질문을 던지지 않는다 — 읽고 판단만 |
| **보안/버그** | 코드 수정 후 머지 (추가 토론 없이) |
| **스타일/취향** | 채택 or 기각 근거 한 줄 남기고 머지 |
| **응답 타임아웃** | 봇 응답이 2분 내 안 오면 기존 리뷰만으로 판단 |

## 토론 가능한 봇

| 봇 | 토론 방법 | 예시 |
|----|---------|------|
| CodeRabbit | `@coderabbitai` 멘션 | "@coderabbitai 이 패턴이 더 안전하지 않나요?" |
| Qodo | `/ask 질문` | "/ask zustand의 함수형 업데이트가 여기서 필요한가요?" |
| Copilot | ❌ 불가 | 일방향 리뷰만 |
| Gemini | ❌ 불가 | 일방향 리뷰만 |

## 리뷰 종합 코멘트 필수 요소

1. **봇 간 동의 사항** — 테이블로 정리
2. **의견 충돌** — 어떤 봇이 뭘 다르게 말했는지
3. **채택/기각 판단** — 각 지적에 대한 판단과 근거
4. **기획 리뷰** — PRD 일치 여부, 서비스 방향성
5. **@coderabbitai 질문 + /ask 질문** — 반드시 같은 코멘트에서 동시에

## 최종 판단 코멘트 필수 요소

1. 봇 응답 요약 (한 줄씩)
2. 최종 판단 (채택/기각)
3. 코드 수정 여부
4. 머지 선언
